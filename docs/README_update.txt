
OKO – локальный MVP co-browsing (Socket.IO) — Обновление

Что изменилось:
- Сервер теперь хранит текущий URL клиента и сразу отправляет его оператору при подключении.
- SDK поддерживает маскировку полей: элементы, которые совпадают с селекторами maskSelectors (по умолчанию input[type=password], а в демо — [data-mask]), игнорируют удалённый ввод и клики.
- В демо поле "Название обоев" помечено data-mask для проверки.

Проверка:
1) npm run dev
2) Откройте http://localhost:3000/demo и http://localhost:3000/operator
3) Нажмите «Поделиться страницей», введите код в операторе.
4) В операторе должна отображаться строка "URL клиента: http://localhost:3000/demo"
5) Проверьте, что оператор может кликать и вводить в поле "Количество", но НЕ может вводить в поле "Название обоев" (data-mask).
6) Нажмите «Стоп» в баннере — сессия должна завершиться.

Дальше:

- Добавить визуальную обратную связь оператору (мини-превью страницы клиента).
