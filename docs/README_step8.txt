
OKO – шаг 8: нормализованные координаты и прокрутка
- Курсор/клик теперь передаются в НОРМАЛИЗОВАННЫХ координатах (0..1) относительно окна браузера оператора.
  На стороне клиента координаты масштабируются под его окно — клики попадают в те же места при разных размерах/мониторах.
- Прокрутка по доле документа (scroll ratio), а не дельтой колеса.
- Сохранены фиксы фокуса и ввода чисел из шага 7.

Порядок проверки:
1) npm run dev
2) /demo + /operator, подключиться по коду.
3) Наведите мышь на поле "Количество" у себя и кликните — у клиента должно фокусироваться именно это поле (даже при разных размерах окон/мониторах).
4) Печатайте цифры и используйте Backspace — ввод должен работать предсказуемо.
5) Прокрутите страницу оператора — у клиента должна синхронно меняться позиция прокрутки.
