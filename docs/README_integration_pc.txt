
OKO — Интеграция в CMS (ПК-версия, без замены текущих файлов)
==============================================================

Минимум нужно подключить 2 скрипта на страницах, где хотите помощь оператора:
  1) /public/sdk.js        — логика совместного управления
  2) /public/oko-widget.js — плавающая кнопка «Помощь на странице» + модалка с кодом

И пометить ключевые элементы атрибутами data-oko-id (например: qty, cart).

I. Bitrix (рекомендуемый путь)
------------------------------
1) Скопируйте файлы sdk.js и oko-widget.js на боевой сайт (например в /local/oko/).
   На локалке эти файлы у вас уже есть в проекте C:\projects\oko\public\
   На проде используйте URL вашего домена:
     https://<ваш-домен>/local/oko/sdk.js
     https://<ваш-домен>/local/oko/oko-widget.js

2) Вставьте теги перед </head> в файле:
   /local/templates/<ВАШ_ШАБЛОН>/header.php

   (см. fragments/bitrix_header_inject.html)

3) Ничего в footer вставлять не обязательно — oko-widget.js сам рисует
   плавающую кнопку «Помощь на странице». Если хотите свою кнопку в конкретном
   месте — возьмите пример из fragments/bitrix_footer_widget_button.html и вставьте
   в footer.php или в шаблон карточки товара.

4) Пометьте ключевые элементы карточки товара атрибутами data-oko-id:
   - Поле количества:  data-oko-id="qty"
   - Кнопка «В корзину»: data-oko-id="cart"
   - Любые другие (hero, search, size, color, footer) по желанию.

   Для Bitrix пример правки дан в fragments/product_markers_bitrix.php —
   это не полный шаблон, а подсказка, как добавить атрибуты к существующим тегам.

5) Страница оператора.
   Вариант А (просто): используйте уже готовую локальную панель по адресу
     http://localhost:3000/operator
   Вариант Б (на сайте): загрузите файл operator/index.html (из вашего проекта)
     на путь /operator/ и откройте в браузере.

II. Любая другая CMS
--------------------
1) Подключите скрипты в <head> шаблона сайта (см. fragments/cms_head_inject.html).
2) (Опционально) добавьте кнопку в удобном месте (см. fragments/cms_inline_button.html).
3) Добавьте атрибуты data-oko-id на элементы карточки товара (см. fragments/markers_examples.html).

III. Тексты для клиента
-----------------------
Готовый блок «о безопасности» — fragments/security_notice.html.
Разместите рядом с кнопкой «Помощь», либо в модалке, либо в FAQ.

IV. Быстрый чек-лист
--------------------
- [ ] Подключены sdk.js и oko-widget.js на странице товара
- [ ] У элементов есть data-oko-id (qty, cart; при желании hero, search и т.д.)
- [ ] Проверен сценарий: клиент нажимает «Помощь» → называет код → оператор вводит код
- [ ] Курсор/клик/ввод/скролл работают
- [ ] Подсказки клиента видны у оператора
- [ ] Поля с секретами помечены data-mask или type="password"
