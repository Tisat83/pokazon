
OKO – локальный MVP co-browsing (Socket.IO)

Содержимое:
- server/index.js — Express + Socket.IO сервер, управление сессиями, трансляция событий.
- public/sdk.js — встраиваемый SDK для сайта, создаёт сессию, принимает команды.
- demo/index.html — демо-страница «клиента» с кнопкой «Поделиться страницей».
- operator/index.html — панель оператора (ввод кода, управление).

Как запустить (локально):
1) Установите зависимости (уже сделано ранее): npm i
2) Запуск сервера: npm run dev
3) Откройте в браузере:
   - Клиент:  http://localhost:3000/demo
   - Оператор: http://localhost:3000/operator
4) На демо-странице нажмите «Поделиться страницей» и скопируйте код.
5) В панели оператора введите код. Двигайте мышь/колесо — на стороне клиента будет двигаться «призрачный» курсор и скролл.

Ограничения MVP:
- Оператор не видит экран клиента, только управляет (визуализацию добавим позже).
- Селектор пути для кликов/ввода может не совпадать, если DOM сильно различается (на MVP тестируем на одной и той же странице).
- Нет TURN/WebRTC (пока достаточно WebSocket).

Планы следующих шагов:
- Добавить захват/стриминг clip-скриншотов или DOM-патчинг для визуальной обратной связи.
- Стабилизировать селекторы (data-атрибуты, устойчивые якоря).
- Ограничения/маскировка полей (data-атрибуты), роли, логирование.
- Встроить в Битрикс через подключение /public/sdk.js в шаблон и кнопку startSession().
